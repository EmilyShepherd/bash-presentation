What is a Shell?

  • A shell is a user's entry point into a computer
  • It "wraps" up the system's functionality
  • Typically come in two forms:
    • Command Line Interpreter (CLI)
    • Graphical User Interface (GUI)

What is the CLI

  • "Interprets" user input, to perform a command
  • Has a very simplistic, syntax
    • Primary function is just to run programs on the machine
    • Very limited logical / conditional operations
    • Scripting is not programming

Bash, the Bourne Again Shell

  • One of the earliest shells, sh, was developed by Stephen Bourne back in 1977
  • Bourne Again Shell (BASH), a successor to sh, released in 1989
  • Systems almost always have sh and bash still on them, or some form of backwards compatible replacement of them

Using Bash

  • Two modes:
    • Interactive
      • Commands are read directly from the user input
    • Non-Interactive
      • Commands are read from a file, called a script
  • The syntax in both modes is the same

Calling Commands

  • Each of our examples is a program: ls, cd, echo
  • When calling commands, you can pass "extra" information as arguments:
    • ls -lah /project/myFolder/myOtherFile.txt
    • Command: ls
    • Arguments: -lah /project/myFolder/myOtherFile.txt

Arguments

  • Arguments are separated by spaces
    • If you need a space in an argument, you can escape it with a backslash, or surround it with quotes:
      • command "argument 1" argumment\ 2
      • command "arg"umen't 1'

Program inputs and output

  • As well as arguments, programs can accept an input stream of data, known as standard input (stdin)
  • Programs can output a stream of data, standard output (stdout)
  • Programs can output a stream of error messages, standard error (stderr)
  • Programs give a numerical status code when they finish, indicating how it went
    • 0 means success
    • Anything other than 0 means failure

More Complex Syntax

  • Sometimes we want to join commands together to perform more complex tasks.
  • A program's stdout doesn't have to just be outputted to the terminal screen. Instead we can direct it elsewhere:
    • Into a file, this is known as a redirect.
    • Into another command, known as a pipe.
